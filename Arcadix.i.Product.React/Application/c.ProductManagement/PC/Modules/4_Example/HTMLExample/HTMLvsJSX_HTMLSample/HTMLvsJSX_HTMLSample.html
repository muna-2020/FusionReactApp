HTML Vs JSX Differences

Here are the main differences that you should be aware of -

You can use Javascript variables directly in your JSX code

If you were writing vanilla Javascript and wanted to insert some variable into an HTML element you would have to do something like this:

                    document.querySelector('#root').textContent = myVariable;

But in JSX, we can use it inside JSX by wrapping it in curly braces {}. This tells the compiler that whatever is inside the curly brackets is a Javascript variable. So now, we can simply put our variable into the markup wherever we see fit, like so:

                    <div id="root">{myVariable}</div>

In the same way, we can also call functions within the curly brackets.

                    <div id="root>{myFunction()}</div>
Adding custom styling to an element uses different syntax

Inline styles are a little different. Take a look at the following HTML and its JSX equivalent.

HTML 
                    <input style='border: 1px solid black; border-radius: 4px'/>
JSX 
                    <input style={{border: '1px solid black', borderRadius: '4px'}}/>
Some attribute names are different in JSX.

Sometimes, if you treat your JSX as HTML, you will encounter warnings about using certain attributes. For example, in JSX, you are supposed to use the attribute ‘className’ instead of ‘class’. So 

 <div class="something"></div>
should be 
<div className="something"></div>
This is also the case with the ‘for’ attribute. In JSX you should use ‘htmlFor’ instead.
